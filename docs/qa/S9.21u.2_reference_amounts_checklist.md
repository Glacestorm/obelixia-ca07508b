# S9.21u.2 — QA: Importes de convenio como referencia (doble carril SafeMode)

## Modelo de producto

Doble carril: cuando SafeMode está activo,

- **Carril final de nómina** queda bloqueado: BASE = 0, PLUS_CONV = 0,
  MEJORA_VOL = 0, conceptos dinámicos de convenio = 0. Calcular / Guardar /
  Preview / PDF no producen un cálculo fiable mientras el contrato no se
  corrija.
- **Carril informativo** muestra los importes seguros del convenio (los que
  proceden literalmente de la tabla salarial) como **referencia no aplicada**,
  para que el operador entienda el contexto y decida cómo corregir el contrato.

## Riesgos residuales conocidos

- La tarjeta de referencia depende de `agreementResolution.tableEntry`. Si
  el resolver no devuelve `tableEntry` por motivos no relacionados con el
  contrato (p. ej. tabla retirada del convenio), la tarjeta no se muestra
  y el operador solo ve el bloque ámbar genérico.
- La tarjeta no resuelve el problema de fondo (contrato incoherente). Sigue
  requiriéndose una corrección manual en People → Empleado → Contrato.
- Los importes mostrados como referencia se obtienen del estado del
  cliente (`agreementResolution`), no se vuelven a fetchear; son los mismos
  que el resolver calculó al cargar la nómina y se respetan sin tocar
  motores ni BD.
