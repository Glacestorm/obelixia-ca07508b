# S9.21u.2 — QA: Importes de convenio como referencia (doble carril SafeMode)

## Modelo de producto

Doble carril: cuando SafeMode está activo,

- **Carril final de nómina** queda bloqueado: BASE = 0, PLUS_CONV = 0,
  MEJORA_VOL = 0, conceptos dinámicos de convenio = 0. Calcular / Guardar /
  Preview / PDF no producen un cálculo fiable mientras el contrato no se
  corrija.
- **Carril informativo** muestra los importes seguros del convenio (los que
  proceden literalmente de la tabla salarial) como **referencia no aplicada**,
  para que el operador entienda el contexto y decida cómo corregir el contrato.

## Caso de prueba — Carlos Ruiz Martín

Empleado con:
- Convenio asignado en `hr_es_employee_labor_data`: Convenio Colectivo Estatal
  de la Industria de la Alimentación y Bebidas.
- Grupo profesional: Grupo III.
- Tabla salarial encontrada para 04/2026.
- Contrato incoherente (`base_salary = 42000` y `annual_salary = 42000`,
  divisor 14, `salary_amount_unit` ambiguo o nulo).

## Otros casos

| Caso | Esperado |
|---|---|
| Empleado coherente (base=2200, divisor 14) con convenio | SafeMode inactivo. No aparece tarjeta de referencia. Importes reales en campos finales. |
| Convenio sin tabla salarial encontrada | SafeMode activo. Tarjeta de identidad visible. Tarjeta de referencia **no** visible (no hay importes de tabla). |
| Empleado sin convenio | SafeMode activo. Bloque "Sin convenio aplicable" visible. Sin tarjetas de identidad ni referencia. |
| Empleado con contrato incoherente + incidencias del periodo | SafeMode activo. Tarjeta de referencia visible. Las incidencias siguen recogidas pero no producen cálculo fiable hasta resolver contrato. |
| Concepto dinámico de convenio con importe 0 | No aparece en la tarjeta de referencia (filtro `amount > 0`). |
| Concepto dinámico porcentual del convenio | No aparece en la tarjeta de referencia (filtro `!isPercentage`) — depende del salario contractual ambiguo. |

## Riesgos residuales conocidos

- La tarjeta de referencia depende de `agreementResolution.tableEntry`. Si
  el resolver no devuelve `tableEntry` por motivos no relacionados con el
  contrato (p. ej. tabla retirada del convenio), la tarjeta no se muestra
  y el operador solo ve el bloque ámbar genérico.
- La tarjeta no resuelve el problema de fondo (contrato incoherente). Sigue
  requiriéndose una corrección manual en People → Empleado → Contrato.
- Los importes mostrados como referencia se obtienen del estado del
  cliente (`agreementResolution`), no se vuelven a fetchear; son los mismos
  que el resolver calculó al cargar la nómina y se respetan sin tocar
  motores ni BD.
